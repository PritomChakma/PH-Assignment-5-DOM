What is Document Object Model (DOM) ?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage as a tree of objects, where each object corresponds to a part of the document. The DOM allows programming languages like JavaScript to interact with and manipulate the content, structure, and style of a webpage.
Here’s a breakdown:
1. Document: Refers to the webpage (HTML document).
2. Object: Each element, attribute, and piece of text in the HTML document is represented as an object (e.g., headings, paragraphs, divs).
3. Model: The objects are organized in a structured way, typically as a tree, where each node represents part of the document.

How Do you Select Element From The DOM ?

To select elements from the DOM in JavaScript, you can use several methods. Here's a simple overview:
1. getElementById() What it does: Selects one element by its unique id.
Example: document.getElementById("header");
Use: When you want a specific element with a known id.
2. getElementsByClassName()
What it does: Selects all elements with a specific class. It returns a list (HTMLCollection).
Example: document.getElementsByClassName("menu-item");
Use: When selecting multiple elements with the same class.
3. getElementsByTagName()
What it does: Selects all elements with a specific tag (like div, p, etc.).
Example: document.getElementsByTagName("p");
Use: When selecting all elements of a specific type.
4. querySelector()
What it does: Selects the first element that matches a CSS selector (like #id, .class, or tag).
Example: document.querySelector(".menu-item");
Use: For more flexible and precise selection of one element.
5. querySelectorAll()
What it does: Selects all elements that match a CSS selector. It returns a list (NodeList).
Example: document.querySelectorAll("div.box");
Use: When you need to select multiple elements using complex CSS rules.

What is Event Delegetion in the contex of the DOM and why is it usefull ?

Event delegation is a technique in JavaScript that allows you to handle events at a higher level in the DOM, instead of attaching event listeners to specific child elements. This is especially useful when you have many similar elements or dynamically added elements that need event handling.
How Event Delegation Works:
Instead of adding event listeners to individual child elements, you attach a single event listener to a common parent element. When an event occurs, it "bubbles" up through the DOM (event bubbling), and you can capture that event at the parent level. From there, you can check if the event originated from a specific child element and handle it accordingly.
Why Event Delegation is Useful:
1.Fewer Event Listeners: Instead of attaching a listener to every element (which can be inefficient), you can use one listener at the parent level. This reduces memory usage and improves performance.
2. Handles Dynamic Content: Event delegation automatically works for new elements added to the DOM. If you dynamically add more li elements to the list, the single event listener on the will still ul work for them without needing any extra code.
3.Simpler Code:It’s easier to maintain one event listener on a parent element rather than many listeners on individual child elements. This simplifies your code and makes it more maintainable.
4.Performance:When dealing with many elements (like in long lists or tables), using individual event listeners can become slow. Event delegation ensures the event is handled efficiently with a single listener.

How do You Manipulate am Elements Attributes and style using the DOM

To manipulate an element’s attributes and style using the DOM in JavaScript, follow these key steps:
1. Access the Element: Use methods like document.getElementById(), querySelector(), or getElementsByClassName() to select the element you want to manipulate.
2. Manipulating Attributes: Use setAttribute() to add or change an attribute (e.g., element.setAttribute('src', 'image.jpg')).
Use getAttribute() to retrieve an attribute’s value.
Use removeAttribute() to delete an attribute.
3. Manipulating Styles: Access the element’s style property (e.g., element.style.color = 'blue';) to modify its inline CSS styles.
You can change multiple properties, like element.style.backgroundColor = 'red'; or element.style.fontSize = '20px';.
